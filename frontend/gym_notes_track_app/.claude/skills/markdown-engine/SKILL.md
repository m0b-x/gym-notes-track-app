# Markdown Engine & Editor

Related docs: [docs/markdown-feature-ideas.md](../../../docs/markdown-feature-ideas.md) (candidate features + fit analysis), [docs/tag-system-roadmap.md](../../../docs/tag-system-roadmap.md) (tag feature plan), and [docs/live-markdown-editor-roadmap.md](../../../docs/live-markdown-editor-roadmap.md) (live editor rendering — status, decision log, remaining gaps). The full invariant list lives in [COPILOT_CONTEXT.md](../../../COPILOT_CONTEXT.md) sections "Markdown Preview Pipeline", "Markdown Block Model And Chunking", "Markdown Lists", and "Ghost Text".

## The one hard constraint

**Every rendered span keeps its exact source offset.** The recursive inline parser (`_parseInline`) routes every leaf text run through `_applyHighlighting` with its exact source offset so search highlights and editor↔preview scroll mapping survive nested emphasis, links, escaping, ghosts, tags, and lists. When adding inline syntax, never break this offset threading.

## Preview pipeline layering (keep intact)

```
OptimizedNoteEditorPage
  -> MarkdownPreviewBloc -> MarkdownRenderService -> LineBasedMarkdownBuilder
       -> MarkdownChunker (block scan + chunk layout)
       -> MarkdownListSyntax (shared list grammar)
  -> MarkdownPreviewBlocView -> SourceMappedMarkdownView
```

- Bloc state is `Equatable`, primitives + `renderHandle: int` only. **Never put `InlineSpan` trees or builders in state** — widgets pull spans from `bloc.renderService.builder` on demand; heavy rebuilds are gated by `buildWhen` on `renderHandle` / `linesPerChunk` / `fontSize`.
- Content sync: `bloc.bindContentProvider(() => controller.text)` once in `initState`; `bloc.markContentDirty()` per keystroke; dispatch `PreviewContentRefreshRequested` for lazy/debounced refresh (500 ms, gated on `!_isLoading` and `state.hasTheme`); `PreviewContentChanged` only for eager pushes (toggle, checkbox, locale change, load). Never dispatch from `build()`.
- Theme dispatch (`PreviewThemeChanged`) only from `MarkdownPreviewBlocView` lifecycle hooks — never `build()`. Scroll progress bypasses the event queue via `bloc.scrollController.updateProgress(progress)`.
- Search sync: call `_searchController.updateContent(content)` from `_pushPreviewContent` / `_scheduleLivePreviewRefresh` when searching — never from `build()`.
- The page owns `final GlobalKey<SourceMappedMarkdownViewState> _previewViewKey` as a named field, bound via `scrollController.bindView(_previewViewKey)` and passed as `viewKey:` — never an inline anonymous key.
- Chunks are **block-aligned with variable line counts** — never compute `lineIndex ~/ linesPerChunk`; use `chunkStartLine(i)`, `chunkIndexForLine(line)`, `MarkdownRenderService.chunkStartLineForLine(line)`.
- Toolbar: reuse `_buildMarkdownBar({required bool enabled})` for both loading and loaded paths — never duplicate the `MarkdownBar(...)` tree.
- Preview links: validate schemes (`http`, `https`, `mailto`, `tel`) before `launchUrl` in `LaunchMode.externalApplication`; localized errors via `CustomSnackbar.showError`.

## Single-source-of-truth grammars (never fork them)

- `MarkdownChunker` (`lib/utils/markdown_chunker.dart`) — block model + chunk layout, shared by preview and the editor debug overlay. Multi-line blocks only (code fences, callouts); single-line content is implicit. New single-widget block types set `atomic: true`.
- `MarkdownListSyntax` (`lib/utils/markdown_list_syntax.dart`) — list grammar shared by editor (`MarkdownListUtils`: Enter-continuation, Tab/Shift-Tab) and renderer. **Never reintroduce a second list regex** — extend `MarkdownListSyntax`.
- `GhostText` (`lib/utils/ghost_text.dart`) — `{{ … }}` placeholder scanning, consumed by preview renderer, editor span builder, tap handling, and the `default_ghost` shortcut. Markers are concealed in the editor (transparent + ~0 width) so caret offsets never desync; tapping selects the run, nothing is mutated.
- `MarkdownCalloutSyntax` (`lib/utils/markdown_callout_syntax.dart`) — `> [!TIP]`-style callouts (types: note, tip, important, warning, caution, success, pr).
- `MarkdownTagSyntax` (`lib/utils/markdown_tag_syntax.dart`) — `#tag` scanning (Unicode-aware, letter-led, word-boundary rules), shared by preview renderer and editor span builder. Tags render tinted; preview tap routes to global search with `#` preserved by `normalizeForSearch`; editor rendering is tint-only (no tap). `TagIndex` roadmap is in the tag roadmap doc.
- `MarkdownMoneySyntax` (`lib/utils/markdown_money_syntax.dart`) — money-ledger grammar + fixed-point arithmetic (`$= $+ $- $* $/` ops, `$$` total, `$?` net change since last `$=`, `$!` target, `$^ N` diff across the last N balance-changing entries — `$=`/`$+`/`$-`/`$*`/`$/`, not display rows — clamped to the current `$=` period (N optional, 1–3 digits, default 1); all display values through `displayValue(m, balance, history, periodStart)` where `history` is the append-only entry-balance list (index 0 = note start, one per entry via `isEntryKind`) and `periodStart` is the current `$=`'s history index), shared by the preview (`_buildMoneyLine` + the `prepare` ledger fold), the editor span builder (tinted ops in the text-keyed cache; `$$`/`$?`/`$^` positional with the value in the key, painted via a fork `CodeInlinePaintSpan`), and `MarkdownEditorLineIndex`'s incremental money pass (`moneyValueAt`, resume state = balance + entry-history length + period-start index). Two shape-only composable prefixes: `#{1,6} ` heading (row renders header-sized; probes are `leadsWithMoney`/`leadsWithMoneyInRange`, which accept it; a `#…$` line failing the money parse falls through to heading rendering) and a letter-led `name:` accent token after the op (`$+ blue: 250`) that overrides the semantic accent when it resolves against the colour palette — unresolved tokens render literally and the line still counts, so the ledger never depends on any setting. The **whole row incl. the label** takes the row accent (`labelStyle`, colour only — base weight kept), semantic sign colour included: plain `$+ 12.50 rent` renders "rent" green, `$- 8 food` renders "food" red. Plus **label-first op rows** (`$= $+ $- $* $/`): the typed amount may trail behind a required `:` (`$= Net worth: 5000`, `## $- blue: Loss: 5000`) — the colon keeps it unambiguous (`$- food 2024` must stay a non-match), the whole tail after the last `:` must be the amount and nothing else, and this is the ONLY shape where `labelStart < amountStart`, so renderers compare them for order and must use `labelEnd` instead of assuming the label runs to the line end. Plus a **value slot**: a lone space-delimited `$` in the label (`$$ Current sum: $`, `$+ 12.50 now $`) renders the row's computed value there instead of its default position, so text can precede the number while the marker stays line-led. Scanned by `_scanValueSlot` **only from inside `parse`**, never from the probes, so non-money lines pay nothing; display-only (balances are identical with and without slots); it moves any row onto the editor's positional path (gate: `valueSlot >= 0 || total/delta/diff`) and the label's `$` becomes the 1:1 paint placeholder — keep any marker substitution one code unit wide (`Δ=` narrows to `Δ`) or the caret drifts. `$=` shows a value only via a slot. Balances are derived from content — never persisted, never mutate counters. Fence lines are inert and the 4096 `maxLineLength` guard matches the editor's raw-render guard so the two ledgers agree. Palette: `MarkdownConstants.moneyPositive/moneyNegative/moneyNeutral`. Display config (global start balance + currency, per-note overridable) threads via `PreviewMoneyConfigChanged` → `MarkdownRenderService` rebuild key (BOTH prepareWithStyle call sites — bloc `_emitPrepared` AND `SourceMappedMarkdownView._buildCache` — must pass identical values) and `MarkdownEditorSpanBuilder.configureMoney` (clears span memos + reseeds the index). `$$`/`$?`/`$^` taps open `MoneyDetailSheet` (preview: recognizer on the pill; editor: wrapper tap-interceptor zone `[markerStart, labelStart)`); entries come from `MarkdownMoneySyntax.collectEntries` (returns `anchorLine` + `prevAnchorLine`). Never route money config through `PreviewThemeChanged` — its (brightness, debugEnabled) dedupe gates would swallow it. See COPILOT_CONTEXT.md "Money Ledger" for the full contract.
- `MarkdownColorSyntax` / `MarkdownColorPalette` (`lib/utils/markdown_color_syntax.dart`) — colour grammar + palette, shared by the preview and the editor span builder. `{name:text}` colours text; `==name:text==` tints a highlight (plain `==text==` keeps the legacy amber, which *is* the `yellow` preset). Names are lowercase `[a-z0-9_-]` (≤24); **an unresolved name is never consumed**, so `{note:x}` and `==note: x==` keep their source text. Presets are light/dark pairs; custom colours (settings, `SettingsKeys.markdownCustomColors`, `name=aarrggbb;…`) shadow presets and are contrast-resolved **once at palette construction** — render time is one map probe, never contrast maths per span. The palette is value-equal on its persisted `source`, which is what makes it a valid render-cache key; thread it exactly like the money config (BOTH `prepareWithStyle` call sites + `configureColors` on the editor builder, which clears both span memos). Editor rendering is pure conceal (`{name:` / `}` transparent + ~0 size — no substitution, no code-unit change). Ghosts win: the editor's ghost-skip branch must **fall through** for a non-ghost `{`, or coloured text stops rendering on every line containing a ghost. See COPILOT_CONTEXT.md "Markdown Colors".
- `MarkdownLinkPatterns` (`lib/utils/markdown_link_patterns.dart`) — link grammar: `bareUrl` / `matchBareUrlEnd` (GFM trailing-punctuation trim) and `matchInlineLinkAt` (`[text](url)` construct), shared by preview renderer, paste line-breaker, editor span builder, and the wrapper's tap zones. Editor `[text](url)` links conceal `[` + `](url)` off-caret; tapping a concealed link opens it via the page's scheme-validated opener, while bare URLs stay tap-to-edit (fully visible source). `![image](url)` stays raw in the editor.
- `MarkdownConstants.calloutAccent` — per-type callout accent palette (light/dark), shared by the preview's callout rendering and the editor's lead-line bar + `[!TYPE]` token tint.

## Live editor rendering (`lib/utils/markdown_editor_span_builder.dart`)

`MarkdownEditorSpanBuilder` renders essential markdown *inside* the re_editor text mode, gated by the "live markdown rendering" editor setting (`SettingsKeys.liveMarkdownRendering`, default true; OFF = plain text + ghost-only rendering). Routing lives in the page's `_buildEditorSpan`: markdown builder first, `_buildGhostEditorSpan` fallback. Invariants:

- **Code-unit preservation** — the returned span always contains exactly the source line's UTF-16 code units. Markers are concealed (transparent + ~0.01 fontSize) or substituted 1:1 (`-`→`•`, `[`→MaterialIcons check glyph); never insert or remove characters, or caret/selection/search offsets desync.
- **Span memo LRU** — built spans are cached per line text and returned as identical instances; the cache is cleared when style/baseColor/primary change and **must only be consulted after the fence check** (fence status is positional, not textual). Reveal (caret/selection) lines bypass the cache.
- **Caret-line reveal** — lines covered by the selection show raw dimmed markers; only marker conceal/substitution may differ between reveal states, never line height (no vertical layout shift on caret moves).
- **Ghosts compose** — ghost regions are opaque to the inline scanner (no run opens or closes inside one) and all content is emitted through the ghost-aware `_emit`. Grammar comes from `MarkdownListSyntax` / `GhostText` — never fork them.
- Tappable checkboxes and concealed links go through the fork's `CodeEditorTapInterceptor` (built in `modern_editor_wrapper.dart`): the tap is claimed at tap-down inside the editor's gesture pipeline, so the caret never moves, focus is never requested (no keyboard rise on an unfocused editor), and re-taps on the same spot fire every time. Zones resolve from line text via the shared grammars, re-resolved at tap-up. Pass-through rules (keep ALL of them): reveal (selection-covered) lines, fence lines, lines over `MarkdownEditorSpanBuilder.maxStyledLineLength`, taps clamped to the line-end offset (blank space), link-construct boundary offsets, escaped/image opens, brackets inside inline-code runs, and **ghosts win** — taps inside a ghost run ride the selection change. Desktop: primary-button only, shift-click passes through, claims are per-pointer, and the outer `ensureInput` is suppressed for claimed taps. A box tap toggles as **one atomic value change with the selection untouched**.
- List lines with content return a `CodeHangingTextSpan` root (fork class): the paragraph provider lays out the marker prefix and content as two paragraphs so wrapped lines get a hanging indent. Never change a line's code units to achieve indent; the hanging span only tags the root.
- Ghost taps are modal: the first tap selects the whole run (typing replaces), a second tap on the same unmodified ghost leaves the collapsed caret in place for in-place editing (engagement state clears when the selection leaves the run or the line's text changes).

## re_editor fork (`packages/re_editor/`)

Local perf-tuned fork — fix bugs in place but avoid API breaks. Preserve: `CodeLines.asString` 2-slot round-robin cache; `_hashCache` short-circuit comparisons; binary-search paragraph lookups in `_code_field.dart`; bounded LRU `_CodeParagraphCache` (512); 50 ms highlight debounce; new `CodeLines` mutation paths must call `cloneShallowDirty()` (never the public constructor). Fork additions to keep intact: `CodeEditorTapInterceptor` (tap-down claim / tap-up action in `_code_selection.dart`, non-mutating `positionAt` in `_code_field.dart`); `CodeHangingTextSpan` → `_HangingParagraphImpl` (two-paragraph hanging indent; all geometry queries must keep mapping piecewise through marker/content); `CodeInlinePaintSpan` (placeholder run + custom canvas paint after `drawParagraph` — exactly one U+FFFC code unit, substituted 1:1 for one source char, height kept under the strut height; the app's `_EditorCheckboxSpan` paints the task checkbox with it, whose indeterminate facet is positional and driven by the span builder's task index alongside the fence index); `CodeDecoratedTextSpan`/`CodeTextDecoration` (rounded chip painted BEFORE the text — tag pills, inline-code chips; reserves no layout space, and `trucate`/`_dropPrefix` must keep untouched subtrees by identity so these spans survive the hanging marker/content split); and the render's text-field semantics (`describeSemanticsConfiguration` + `markNeedsSemanticsUpdate` in the codes/hasFocus/readOnly setters — the editor's only screen-reader surface). Paste flows through the app's `ListAwarePasteController` (`lib/utils/list_aware_paste.dart`, a `CodeLineEditingControllerDelegate`) — list-aware multi-line paste; IME-key paste bypasses it by design. Per-line height: `_CodeParagraphProvider` gives a paragraph whose **root span** sets a fontSize different from the base style its own strut + preferred line height (memoized per size) — this is how editor headers render taller; don't key line-height decisions on anything but the root span's fontSize. Anything anchored to a specific line (caret, selection handles, IME caret rect, toolbar anchors, floating cursor) must use the render's `lineHeightOfLine(index)` / `lineHeightAtOffset(offset)` — never the base `lineHeight` getter, which is only for whole-viewport estimates (scroll math, chunk indicators).

## Editor hot-path rules

No per-keystroke string copies, synchronous DB writes, or expensive rebuilds on the typing path. Auto-save (`auto_save_service.dart`) debounce/interval/lifecycle-flush/retry behavior must stay reliable. Adaptive chunk sizing lives in `MarkdownChunker.adaptiveChunkSize` (cap 100), applied by the render service.
